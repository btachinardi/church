## Detection Patterns

1. **Force-try**
   - Grep for `try!` â€” every instance in non-test code is a violation
   - In test code: acceptable for known-good inputs (mark as exempted)

2. **Empty catches**
   - Grep for `catch\s*\{\s*\}` â€” empty catch blocks
   - Grep for `catch\s*\{[^}]*//` â€” catches with only comments (no handling)
   - Check for `catch { _ = error }` or similar no-ops

3. **Bare throws**
   - Grep for `throws\s*[^(]` â€” functions that throw without a type
   - Grep for `throws\s*->` or `throws\s*\{` â€” bare throws before return/body
   - Note: Swift 6 typed throws syntax is `throws(ErrorType)`

4. **Error-discarding try?**
   - Grep for `try\?` â€” check if error is intentionally discarded
   - Each `try?` must have a comment or context justifying why the error is irrelevant

5. **Generic error types**
   - Grep for `Result<.*,\s*Error>` â€” untyped failure
   - Check for `catch let error` without downcasting to specific types
